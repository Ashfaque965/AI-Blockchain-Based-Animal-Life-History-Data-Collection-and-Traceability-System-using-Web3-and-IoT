

### 🤝 Contributing to AnimalChain ###




Thank you for your interest in contributing to **AnimalChain** — a decentralized animal life history and traceability system powered by blockchain, Web3, and IoT.

We welcome contributions from developers, researchers, designers, and enthusiasts of all skill levels.

By participating in this project, you agree to:

## 🛠️ Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub.

---

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/AI-Blockchain-Based-Animal-Life-History-Data-Collection-and-Traceability-System-using-Web3-and-IoT.git
cd AI-Blockchain-Based-Animal-Life-History-Data-Collection-and-Traceability-System-using-Web3-and-IoT
```

---

### 3. Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

---

### 4. Install Dependencies

```bash
npm install
```

---

### 5. Run the Project Locally

```bash
npx hardhat node
npx hardhat run scripts/deploy.js --network localhost
npm run dev
```

---

## 📦 Project Structure

```
contracts/   → Smart contracts (Solidity)
frontend/    → React / Next.js application
backend/     → Node.js APIs (optional)
scripts/     → Deployment scripts
test/        → Smart contract tests
docs/        → Research and documentation
```

---

## ✍️ Commit Guidelines

Please follow clean commit messages:

### ✔ Good Examples

```
feat: add animal registration contract
fix: resolve ownership transfer bug
docs: update README instructions
ui: improve dashboard layout
```

### ❌ Avoid

```
update
fix stuff
new changes
```

---

## 🔄 Pull Request Process

1. Fork the repository
2. Create your feature branch
3. Make your changes
4. Test your code locally
5. Push your branch
6. Open a Pull Request (PR)

### PR Title Format:

```
feat: add new animal medical record module
fix: correct IPFS upload issue
```

---

## 🧪 Testing

Before submitting:

* Smart contracts should compile without errors
* Frontend should run without console errors
* Ensure no breaking changes

Run:

```bash
npx hardhat test
```
